## Overview

Annie Mei provides two utility commands for getting help and checking bot status:

- `/help` - Displays a guide to available commands and usage tips
- `/ping` - Verifies the bot is online and responding

---

## When to Use Each Command

| Scenario | Command | Reason |
|----------|---------|--------|
| First time using the bot | `/help` | Get a full guide to all features |
| Forgot command syntax | `/help` | See all commands with examples |
| Bot seems unresponsive | `/ping` | Quick health check |
| Want to test permissions | `/ping` | Minimal response to verify bot access |
| Onboarding new users | `/help` | Share the embed for reference |

<Tip>
  Both `/help` and `/ping` respond instantly because they don't query external APIs. If either command fails, the bot may be offline or lacking permissions in your server.
</Tip>

## Troubleshooting

<AccordionGroup>
  <Accordion title="/help or /ping doesn't respond">
    **Possible causes:**
    - Bot is offline or restarting
    - Bot lacks permission to send messages in the channel
    - Discord API issues
    
    **Solution:**
    - Check bot's online status in the member list
    - Verify the bot has "Send Messages" and "Use Application Commands" permissions
    - Try again in a few seconds
  </Accordion>

  <Accordion title="/help image doesn't load">
    If the help embed shows a broken thumbnail:
    - The `mei.jpg` file may be missing from the bot's directory
    - This doesn't affect functionality, only the visual appearance
    - The rest of the help embed will still display correctly
  </Accordion>

  <Accordion title="/ping mentions the wrong user">
    This shouldn't happen unless the bot has a critical bug. If `/ping` mentions someone else:
    - Report the issue to the bot administrator
    - Restart the bot
  </Accordion>
</AccordionGroup>
