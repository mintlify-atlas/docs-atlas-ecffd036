# Installation Guide

This guide covers how to self-host Annie Mei by building from source.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Rust 1.93 or higher** - Install from [rustup.rs](https://rustup.rs/)
- **PostgreSQL** - Any recent version (the bot uses Diesel ORM)
- **Redis** - For caching API responses
- **Git** - To clone the repository

<Note>
  Annie Mei uses the Rust 2024 edition. Make sure your Rust installation is up to date:
  ```bash
  rustup update
  ```
</Note>

## Environment Setup

Annie Mei requires several environment variables to function. You'll need accounts and API keys from the following services:

### Required Services

1. **Discord Bot Token** - Create a bot at the [Discord Developer Portal](https://discord.com/developers/applications)
2. **Sentry Account** (optional but recommended) - For error tracking at [sentry.io](https://sentry.io/)
3. **Spotify API Credentials** - Register an application at [Spotify for Developers](https://developer.spotify.com/)

### Environment Variables

Create a `.env` file in the project root with the following variables:

```bash
# Discord
DISCORD_TOKEN=your_discord_bot_token

# Database
DATABASE_URL=postgresql://username:password@localhost/annie_mei

# Cache
REDIS_URL=redis://localhost:6379

# Spotify API
RSPOTIFY_CLIENT_ID=your_spotify_client_id
RSPOTIFY_CLIENT_SECRET=your_spotify_client_secret

# Error Tracking (optional)
SENTRY_DSN=your_sentry_dsn
SENTRY_TRACES_SAMPLE_RATE=0.1

# Environment
ENV=development
```

<Warning>
  Never commit your `.env` file to version control. Add it to `.gitignore` to keep your credentials safe.
</Warning>

## Installation Steps

<Steps>
  <Step title="Clone the Repository">
    Clone the Annie Mei repository from GitHub:
    
    ```bash
    git clone https://github.com/yourusername/annie-mei.git
    cd annie-mei
    ```
  </Step>
  
  <Step title="Install Diesel CLI">
    Install the Diesel CLI tool for database migrations:
    
    ```bash
    cargo install diesel_cli --no-default-features --features postgres
    ```
  </Step>
  
  <Step title="Setup the Database">
    Create the PostgreSQL database and run migrations:
    
    ```bash
    # Create the database
    createdb annie_mei
    
    # Run migrations
    diesel migration run
    ```
    
    <Note>
      If you get a connection error, verify your `DATABASE_URL` in the `.env` file is correct.
    </Note>
  </Step>
  
  <Step title="Build the Project">
    Build Annie Mei in release mode for optimal performance:
    
    ```bash
    cargo build --release
    ```
    
    This will compile the project with optimizations. The first build may take several minutes as it compiles all dependencies.
    
    <Note>
      For development, you can use `cargo build` without the `--release` flag for faster compilation times.
    </Note>
  </Step>
  
  <Step title="Run the Bot">
    Start the bot:
    
    ```bash
    cargo run --release
    ```
    
    You should see log output indicating the bot is connecting to Discord and registering slash commands.
    
    The bot will:
    - Connect to PostgreSQL and run any pending migrations
    - Connect to Redis for caching
    - Connect to Discord and register global slash commands
    - Start listening for interactions
  </Step>
  
  <Step title="Verify Installation">
    Test the bot by running a command in Discord:
    
    ```
    /ping
    ```
    
    If you receive a response with latency information, your installation is successful!
  </Step>
</Steps>

## Development Workflow

If you're planning to contribute or modify Annie Mei:

### Quick Type Checking

```bash
cargo check
```

Runs fast type checking without producing binaries.

### Code Formatting

```bash
cargo fmt
```

Formats code according to Rust style guidelines.

### Linting

```bash
cargo clippy
```

Runs the Clippy linter to catch common mistakes and suggest improvements.

### Running Tests

```bash
cargo test
```

Executes the test suite.

### Database Migrations

To create a new migration:

```bash
diesel migration generate migration_name
```

This creates `up.sql` and `down.sql` files in the migrations directory.

<Warning>
  Never manually edit `src/schema.rs` - it's auto-generated by Diesel when you run migrations.
</Warning>

## Production Deployment

For production deployments:

1. **Use release builds** - Always compile with `--release` for performance
2. **Set appropriate log levels** - Use the `RUST_LOG` environment variable
3. **Configure Sentry** - Set `SENTRY_DSN` and `SENTRY_TRACES_SAMPLE_RATE` for error tracking
4. **Use managed services** - Consider hosted PostgreSQL (e.g., Neon) and Redis (e.g., Upstash) for reliability
5. **Enable TLS** - Ensure Redis and PostgreSQL connections use TLS in production

### Recommended Environment Variables for Production

```bash
ENV=production
RUST_LOG=info,serenity=warn
SENTRY_TRACES_SAMPLE_RATE=0.1
```

## Troubleshooting

### Connection Issues

- **Database connection fails**: Verify PostgreSQL is running and `DATABASE_URL` is correct
- **Redis connection fails**: Verify Redis is running and `REDIS_URL` is correct
- **Discord connection fails**: Check your `DISCORD_TOKEN` is valid and the bot is enabled in the Developer Portal

### Build Issues

- **OpenSSL errors**: The project uses vendored OpenSSL. If you encounter issues, ensure you have build tools installed
- **Diesel schema errors**: Run `diesel migration run` to regenerate the schema

### Runtime Issues

- **Commands not appearing**: Global commands can take up to an hour to propagate. For development, consider using guild-specific commands
- **Slow responses**: Check your Redis cache is working - the bot caches API responses for performance

<Note>
  For more detailed troubleshooting and community support, check the GitHub repository's issues page.
</Note>
